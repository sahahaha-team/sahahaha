### 🚀 [11주차] Feature Flags 및 점진적 배포 실습

**1. Feature Flag 및 A/B 테스트 구현 로직**
* **플래그 1 (`ENABLE_NEW_UI`):** 환경 변수 기반으로 전역적인 신규 기능 노출 여부를 토글합니다.
* **플래그 2 (`BETA_ADMIN_ONLY`):** 대상 사용자 기준(사용자 ID가 'admin'으로 시작하는 경우)으로 특정 권한을 부여하여 베타 기능을 렌더링합니다.
* **플래그 3 (`ENABLE_AB_TEST`):** 사용자의 ID 값을 해싱(Hashing) 처리하여 홀수/짝수로 분류함으로써, 동일한 사용자가 접속할 때마다 항상 같은 Variant(A 또는 B) 화면을 볼 수 있도록 할당 일관성을 보장했습니다.
* **이벤트 추적:** 사용자가 할당된 Variant 화면에서 액션을 취할 때, `userId`와 `variant` 정보가 포함된 로그를 생성하고 수집하는 이벤트 추적 로직을 구현했습니다.

---

**2. [선택과제] Canary 점진적 롤아웃 및 자동 롤백 시나리오 설계**

새로운 기능을 안정적으로 배포하기 위해 다음과 같은 비율의 트래픽 라우팅 및 헬스체크 기반 롤백 시나리오를 구성합니다.

* **Canary 롤아웃 단계 설정:**
  1. **1% 트래픽 배포:** 사내 테스트 및 극소수 외부 사용자 대상 초기 검증.
  2. **10% 트래픽 배포:** 모니터링 대시보드 관찰 및 에러 로그 탐지.
  3. **50% 트래픽 배포:** A/B 테스트 지표 본격 비교 및 부하 테스트 확인.
  4. **100% (GA) 배포:** 성공적인 검증 완료 후 전체 사용자 대상 롤아웃.

* **헬스체크 기반 자동 롤백 (Auto-Rollback) 시나리오:**
  * **조건:** 10주차에 설정한 `/healthz` 엔드포인트를 주기적으로 호출.
  * **트리거:** Health Check 응답 실패 또는 에러율 임계치 초과 시.
  * **동작:** 즉시 Feature Flag 스위치를 Off로 변경하거나 안정 버전으로 가중치를 원복합니다.
