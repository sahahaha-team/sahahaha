# 🚀 3주차 과제: 애자일 스크럼 및 칸반 보드 구축

본 폴더는 GitHub Projects를 활용한 칸반 보드, 마일스톤, 라벨 체계 및 백로그 구성 결과를 정리한 문서입니다.

## 1. 프로젝트 칸반 보드 구성
- **상태 컬럼:** Backlog ➡️ To Do ➡️ In Progress ➡️ Review ➡️ Done
- 총 11개의 핵심 이슈를 도출하여 백로그를 구성하였습니다.

## 2. 마일스톤 (Milestones)
- **🚩 Milestone 1:** 데이터 파이프라인 및 핵심 검색 엔진 구축 (3월 ~ 4월 말)
- **🚩 Milestone 2:** AI 대화 엔진 연동 및 웹 프로토타입 완성 (5월 ~ 6월)

## 3. 라벨 체계 (Labels)
`✨ Feature`, `🐛 Bug`, `📊 Data`, `🤖 AI/RAG`, `🖥️ Backend`, `🎨 Frontend`, `🚨 Urgent`

---

## 📈 (선택과제) Agile Metrics Analysis (Sprint 1 예상치)

프로젝트 진행 중 다음 지표를 추적하여 스프린트 효율성을 개선할 계획입니다.

* **Velocity (벨로시티):** 주당 평균 3~4개의 이슈 완료 목표
* **Cycle Time (사이클 타임):** 평균 3~5일 소요 예상 (데이터 수집 등 시간이 오래 걸리는 작업 위주로 초기 지연 가능성 반영)
* **Burndown Insight:** 마일스톤 마감일에 맞추어 GitHub Insights의 Burn up/down 차트를 활용해 남은 작업량을 시각화하고, 리뷰 지연(Review 병목) 현상을 방지합니다.
