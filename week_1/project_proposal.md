## 📢 프로젝트 배경 및 필요성
- 현재 공공기관 홈페이지는 다양한 행정 및 복지 정보를 제공하지만, 메뉴 구조가 복잡하여 주민들이 원하는 정보를 빠르게 찾기 어렵습니다. 
- 기존의 단순 키워드 검색 방식은 접근성이 떨어지므로, 사용자가 일상적인 언어(자연어)로 질문하면 즉시 정확한 정보를 찾아주는 AI 상담 시스템이 필요합니다.
- 이를 통해 주민들의 정보 접근성을 높이고, 단순/반복적인 민원 문의를 AI가 1차적으로 처리하여 행정 업무의 효율성을 크게 개선할 수 있습니다.

---

## 🎯 문제 정의 (Problem Definition)
1. **복잡한 정보 구조:** 홈페이지 메뉴가 복잡해 사용자가 민원 신청 절차나 복지 정보의 위치를 찾기 어려움.
2. **기존 검색의 한계:** 사용자 의도를 파악하지 못하는 단순 검색 방식으로 인해 정보 접근성 저하.
3. **어려운 행정 용어:** 제공되는 정보가 전문적인 행정 용어로 되어 있어 일반인이 즉각적으로 이해하기 어려움.

---

## ✨ 핵심 기능 (Core Features)
* **비정형 행정 데이터 수집 및 자동 구조화**
  * 사하구청 홈페이지의 데이터를 크롤링하고, LLM을 활용하여 부서, 주제, 키워드 등의 메타데이터를 자동으로 태깅하여 구조화합니다.
* **2단계 하이브리드 검색 시스템 (Hybrid Retrieval)**
  * 1단계로 메타데이터 기반 필터링을 통해 탐색 범위를 좁히고, 2단계로 Vector 기반 의미 검색을 수행하여 사용자 질문과 가장 유사한 문서를 정확하게 찾아냅니다.
* **멀티턴 대화 및 사용자 맞춤형 답변(출처 포함) 제공**
  * 이전 대화의 문맥을 유지하며 모호한 질문에는 역질문을 던지는 멀티턴 대화를 지원합니다.
  * 어려운 행정 용어를 이해하기 쉬운 자연어로 요약하고, 답변의 신뢰성을 위해 근거가 되는 홈페이지 URL(출처)을 함께 제공합니다.

---

## 🛠 기술 스택 (Tech Stack)
* **Language:** Python
* **AI & Search:** LLM 기반 자연어 처리 모델, LangChain, RAG, Vector DB (FAISS 또는 Chroma)
* **Data Processing:** Pandas
* **Backend / Frontend:** FastAPI 기반 비동기 서버, 반응형 웹 인터페이스
* **Tools:** Visual Studio Code, GitHub

## 🗓 16주 마일스톤 (Milestone)
* **1~3주차 (데이터 수집):** 사하구청 홈페이지 행정 정보 크롤링 및 전체 데이터 수집 파이프라인 구축
* **4~6주차 (데이터 구조화):** 증분 크롤링 적용 및 LLM 기반 메타데이터(부서, 주제, 키워드) 자동 생성 태깅 작업
* **7~9주차 (검색 시스템 구축):** 메타데이터 필터링 및 Vector DB 기반 하이브리드 의미 검색 로직 구현
* **10~12주차 (AI 상담 엔진 개발):** 멀티턴 대화 처리 로직 구현, 프롬프트 엔지니어링을 통한 답변 요약 및 출처(URL) 제공 기능 연동
* **13~16주차 (서비스 구현 및 테스트):** FastAPI와 반응형 웹을 연동한 프로토타입 UI/UX 개발, 시스템 검증 및 최종 마무리
